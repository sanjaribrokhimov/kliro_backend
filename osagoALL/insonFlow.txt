Структура данных и обязательные поля

Для создания контракта (Шаг 4 в Сценарии интеграции) не всегда требуется передавать полные данные, если система может подтянуть их автоматически.
 Ниже описан минимальный набор обязательных полей.

1. Общие данные (Авто и Детали)

Этот блок заполняется всегда одинаково, независимо от типа страхователя.

{
  // Детали полиса
  details: {
    startDate: "2026-01-23",     // Дата начала (yyyy-MM-dd)
    period: 12,                  // Срок: 6 или 12 месяцев
    driverNumberRestricted: true // Ограничено ли кол-во водителей (true/false)
  },

  // Данные автомобиля
  vehicle: {
    governmentNumber: "01A777AA", // Госномер
    techPassport: {
      series: "AAF",    // Серия техпаспорта
      number: "1234567" // Номер техпаспорта
    }
  }
}


2. Данные персон (Страхователь и Водители)

Для людей (объекты applicant.person и элементы массива drivers) действует логика разделения на резидентов и нерезидентов. Типы резидентов описаны в Справочниках.

А. Если человек — Резидент Узбекистана (residentType: 1)

Достаточно передать паспортные данные.
 Остальную информацию (ФИО, адрес, дату рождения) система заполнит автоматически на основе баз данных.

{
  residentType: 1, // 1 = Резидент (см. справочник Resident Types)
  passportData: {
    series: "AA",     // Серия паспорта
    number: "1234567" // Номер паспорта
  },
  // Для Страхователя (Applicant) дополнительно обязательно:
  phoneNumber: "998901234567"
}


Б. Если человек — Иностранец (residentType: 2)

Автоматическое заполнение невозможно, требуется ручной ввод всех полей, включая ID региона и района (см. Регионы и Районы).

{
  residentType: 2, // 2 = Иностранец (см. справочник Resident Types)
  passportData: {
    series: "AA",
    number: "1234567"
  },
  // Обязательный блок для иностранца
  fullName: {
    firstname: "John",
    lastname: "Doe",
    middlename: "Smith" // Если отчества нет, можно передать "XXX" или пустую строку
  },
  birthDate: "1990-01-01",

  // ТОЛЬКО для Страхователя (Applicant)-иностранца дополнительно нужны:
  gender: 1,       //  0 - Женский, 1 - Мужской
  address: "Toshkent...",
  regionId: 10,    // ID региона (см. справочник Regions)
  districtId: 1003, // ID района (см. справочник Districts)
  phoneNumber: "998901234567"
}


Создание контракта

Основной метод оформления полиса.
 Поддерживается оформление как ограниченных (с указанием водителей), так и неограниченных полисов (Без ограничений) для физических и юридических лиц.

Base URL: https://testapi-ersp.insonline.uz/
Метод: POST api/v2/osago/contract
Документация: Открыть в Swagger UI
ВАЖНОЕ ПРАВИЛО ВАЛИДАЦИИ ВОДИТЕЛЕЙ
Заполнение массива drivers напрямую зависит от параметра driverNumberRestricted (ограничение количества водителей):

Ограниченный полис (driverNumberRestricted: true):
 Массив drivers обязателен. В нем должен присутствовать хотя бы один объект водителя. Пустой массив вызовет ошибку валидации.
Полис без ограничений (driverNumberRestricted: false):
 Массив drivers заполнять не нужно (он должен быть пустым [] или отсутствовать в запросе).
Данные владельца (Owner)

Поле isOwner указывает, является ли страхователь владельцем автомобиля.

ЕСЛИ СТРАХОВАТЕЛЬ НЕ ВЛАДЕЛЕЦ (ISOWNER: FALSE)
Если параметр isOwner установлен в false, то обязательно требуется передать объект owner.

Структура объекта owner должна быть идентична структуре:

person — если владелец физическое лицо;
organization — если владелец юридическое лицо.

 Данный справочник используется для выбора корректной категории ТС (легковой, грузовой и т.д.) при расчете страховой премии.

Метод: GET https://testapi-proref.insonline.uz/api/v1/reference/vehicle-types
Документация: Открыть в Swagger UI
Пример ответа

{
  "status": null,
  "data": {
    "vehicleTypes": [
      {
        "id": 1,
        "name": "Легковые автомобили"
      },
      {
        "id": 15,
        "name": "Трамваи, мотоциклы и мотороллеры, тракторы, самоходные дорожно-строительные и иные машины"
      },
      {
        "id": 6,
        "name": "Грузовые автомобили"
      },
      {
        "id": 9,
        "name": "Автобусы и микроавтобусы"
      }
    ]
  }
}
